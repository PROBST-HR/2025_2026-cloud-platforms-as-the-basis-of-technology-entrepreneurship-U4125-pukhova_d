# Лабораторная работа №1  
## Обзор Google Cloud и исследование основных сервисов

---

## Цель работы

Ознакомиться с основными возможностями и преимуществами облачной платформы **Google Cloud Platform (GCP)**, а также получить практические навыки работы с сервисами IAM, Compute Engine и Cloud Storage.

---

## Используемые сервисы

- IAM (Identity and Access Management)
- Compute Engine (виртуальные машины)
- Cloud Storage
- gsutil (CLI для работы с Cloud Storage)

---

### 5. Практическая проверка доступа (совместно с преподавателем)

В рамках выполнения лабораторной работы совместно с преподавателем был создан выделенный сервисный аккаунт, которому были назначены роли доступа. После этого была проведена проверка доступов — доступ к Cloud Storage был подтверждён.

Далее роли сервисного аккаунта были изменены, после чего при повторной проверке доступ к ресурсам Cloud Storage был ограничен. На основании проведённого эксперимента был сделан вывод о том, как работают роли и механизмы управления доступом в Google Cloud Platform.

---

### 6. Проверка доступа к Cloud Storage

На VM была выполнена команда:

```bash
gsutil ls gs://lab1-bucket-itmo
```
## 7. Копирование файлов из бакета на VM

Создана локальная папка:

```bash
mkdir lab1-files
```
Выполнено копирование файлов:

```bash
gsutil cp gs://lab1-bucket-itmo/* lab1-files/
```
Проверка:

```bash
ls -lah lab1-files
```
Результат: файлы успешно загружены на виртуальную машину.

## 8. Изменение роли Service Account

Роль сервисного аккаунта была изменена:

- Было: **Storage Admin**  
- Стало: **Compute Viewer**

## 9. Проверка доступа после изменения роли

Повторная попытка доступа:

```bash
gsutil ls gs://lab1-bucket-itmo
```

## Выводы

В ходе лабораторной работы было изучено управление доступами в Google Cloud Platform с использованием IAM.

Было установлено:

- Доступ к ресурсам Google Cloud определяется назначенными IAM ролями сервисного аккаунта.  
- Роль **Storage Admin** предоставляет полный доступ к Cloud Storage.  
- Роль **Compute Viewer** не предоставляет доступ к Cloud Storage.  
- Изменение ролей не всегда применяется мгновенно в активной сессии, так как используются ранее выданные токены доступа.  
- Для полного применения изменений может потребоваться перезапуск виртуальной машины или обновление аутентификации.  
- Совместное выполнение эксперимента с преподавателем позволило наглядно увидеть влияние изменения ролей на доступ к облачным ресурсам и понять принцип работы IAM в Google Cloud.
